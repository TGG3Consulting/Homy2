# Homy — UX/UI аудит и визуальные направления (Фаза 1)

> Каноническое имя бренда — **Homy** (одна «m»). Написание «HomLy» в коде/путях игнорируется.

**Дата:** 7 июля 2026 · **Фокус:** UI/UX по рендеру (не код-ревью, не инженерия).
**Метод:** сайт поднят на `localhost:3000` (реальная БД, Docker), живой проход через браузер + существующие скриншоты состояний с авторизацией. Данные для макетов — реальные листинги из `homly_data.json`.
**Ограничение по мобильному:** истинный вьюпорт 390px снять не удалось — окно браузера зафиксировано на 1920px (`window.innerWidth = 1920` даже после ресайза), эмуляция устройства инструментом недоступна. Мобильный опыт оценён по структуре лейаутов; рекомендуется отдельная проверка на реальном устройстве.

## 1. Инвентарь экранов и флоу

**Публичные:** Лендинг `/` (RU) · Как это работает `/how-it-works` (RU) · Вход `/login` (**EN-only**) · Регистрация `/register` (**EN-only**) · Сравнение `/compare` (пустое состояние, RU) · Каталог `/properties` → **редирект** в `/results`.

**Ядро — AI-поиск:** AI-диалог (уточнения, «Думаю…», «AI анализирует…») · Результаты: карта + список карточек + панель чата · Пустое/0-состояние центра · Детальная карточка (галерея, «Виртуальный тур», «Мнение Homy», Advantages/Consideration, действия, консультант) · Property Intelligence (Developer&legal / Location / Infrastructure / Investment, **EN-only**) · Панель живого чата с консультантом.

**Кабинет брокера:** Обзор/Overview (EN и RU) · Мои просмотры (Upcoming/Completed) · Создать просмотр (календарь + слоты) · Разместить объект `/list-property` → gate на `/login` · сайдбар: Clients / Properties / Messages / Deals / Settings.

**Флоу:** (1) покупатель: лендинг → запрос → AI уточняет → результаты (карта+список) → карточка → консультант/запись; (2) сравнение: 2+ объекта → `/compare`; (3) брокер: вход → дашборд → объявления/клиенты/просмотры; (4) онбординг: регистрация (имя/фамилия/отчество/email/пароль) или Google → вход.

## 2. Топ UI/UX проблем

### 🔴 Дизайн-система и доверие визуала
- **U1. Бейджи не отражают оценку.** «★ Лучший выбор Homy» стоит на всех карточках; в открытой карточке «Мнение Homy» при этом гласит «НЕ подходит». Бейдж-декор, противоречащий тексту рядом, обнуляет доверие. Нужна система статусов: бейдж = следствие оценки.
- **U2. Match-score без смысла.** «30% совпадение» рядом с «Лучший выбор» на всех карточках — непонятно, хорошо это или плохо; нет шкалы/цвета/«из чего складывается».
- **U3. Слабые загрузка/пустота.** Центр результатов — большой пустой серый прямоугольник без скелетона и подсказки. При этом у `/compare` пустое состояние сделано аккуратно — паттерн есть, применён не везде.
- **U4. Форматирование чисел без предохранителя.** Цены выводятся «как есть» (в выдаче встречалось 9-значное число в поле аренды) — компонент цены должен переносить «грязные» значения без потери вёрстки.

### 🟠 Консистентность
- **U5. Разрывы локализации.** Вход, Регистрация и Property Intelligence — **только EN**; остальное RU. Армянские (hy) переводы в данных — **битые заглушки** («Delay delay»). Для рынка Армении язык (hy/ru/en) — часть доверия.
- **U6. Несогласованная навигация.** Лендинг без верхнего меню; `/how-it-works` — с полной шапкой. Нет постоянного глобального навбара и футера.
- **U7. Неустоявшийся бренд.** Имя как Homy / HOMY / HOMLY; логотип на лендинге — мультяшный домик + wordmark, дальше — просто разрядка «HOMY». **Канон — Homy.**
- **U8. Два языка дизайна.** Тёмный зрелый кабинет брокера vs светлый «игрушечный» фронт — ощущение двух продуктов.
- **U9. Инородная фотография.** Типовой западный сток (жёлтое кресло-акцент), повторяется между карточками, без ереванского контекста.

### 🟡 Визуальная система
- **U10. Типографика без характера** (один гротеск, иерархия на размере; геройский заголовок лендинга — бледно-серый, проигрывает логотипу).
- **U11. Цвет/контраст:** один фиолетовый + много почти-белого; все trust-состояния ярко-зелёные («зелёнка» = промо). Контраст низкий, местами ниже WCAG AA.
- **U12. Ритм/spacing:** лендинг вертикально отцентрован с большими пустотами; нижний ряд чипов оторван.
- **U13. Плотность детальной карточки** (фото + текст + действия + консультант в узком окне).
- **U14. Тон «продажника»:** именной консультант с телефоном и «Написать/Позвонить» — на каждом экране; давит лид-геном.

### ⚪ Вне зоны UI — бэкенд/логика (без углубления)
Ответ AI ≠ показанные карточки · фильтр по бюджету/комнатам не соблюдается · недостоверные числа в метриках · аномальные цены. Не задачи редизайна, но их надо закрыть параллельно.

## 3. Поддержка позиционирования (нейтральный слой доверия для покупателя)

**Работает:** Property Intelligence (Developer/legal/Location/Infrastructure/Investment) — визуальный язык независимой проверки; блок «Мнение Homy» с колонкой **Consideration** (показ минусов) — честный паттерн, усиливать. Формулировки-«аналитик» («проанализировано», «маршрут проанализирован»).

**Подрывает:** сплошная «зелёнка/проверено» читается как заверение, а не сбалансированная доказательность; постоянный именной консультант + «★ Лучший выбор» на всём + громкие цифры («350+ сделок») склоняют тон к брокериджу; противоречивые бейджи (U1) бьют по образу честного советника.

**Вывод для редизайна:** архитектура (что показываем) уже поддерживает доверие. Визуалу нужно: (1) сместить тон с «промо-зелёного» на **сбалансированную доказательность** (зелёный/нейтральный/янтарный); (2) сделать бейджи и score следствием оценки; (3) сделать консультанта **опциональной** помощью, а не давлением; (4) выстроить единый визуальный язык, иерархию и типографику.

## 4. Референсы (Chrome + ресёрч)

**frankai.ae** (AI-риелтор, Дубай) — тёмный кинематографичный герой, издательский сериф + оранжевый, персонаж-бренд (Б. Франклин), пресса «285+ publications». *Перенять:* уверенный издательский заголовок, тёмное/кремовое контрастом, один акцент. *Избегать:* бравада «no boss, no delays», культ личности, хайп — противоположность нейтральному доверию Homy.

**propertyfinder.ae** — чистый портал: герой + фильтр-чипы (тип/спальни/цена/удобства/площадь), «поиск по времени в пути» (релевантно commute-анализу Homy), verified-бейджи, аккуратные скелетоны, ясный specs-ряд. *Перенять:* фильтр-чипы, commute-поиск, verified, скелетоны. *Избегать:* агент-центризм («SUPERAGENT», Call/WhatsApp на каждой карточке).

**bayut.com** — доступ закрыт CAPTCHA (не проходил); портал агрегатор-типа, аналогичен PropertyFinder (агент-forward). Вывод переносится по аналогии.

**Аналоги-продукты (веб-ресёрч):** Zillow (Zestimate — оценка+доверие, US) · **PriceHubble** (AI property intelligence, 10 стран; «показывать логику за числами») · **Casafari** (чистая, дедуплицированная, валидированная база, 20+ стран — доверие через качество данных) · **HouseCanary** (AVM/прогнозы, 136M объектов, US) · **PropStream** (AI-ассистент с «comparable sales с прозрачным источником» — близко к чат-модели Homy). **Ключевой тренд 2025–26: доверие через прозрачность — показывать обоснование за цифрами.** Это ровно позиционирование Homy — усиливать визуально.

**Тон travel/fintech:** чистые нейтральные интерфейсы с данными-героями и балансом сигналов (не сплошной зелёный) — Homy должен ощущаться доверительнее и чище, чем брокерские доски.

## 5. Три визуальных направления — «Доказательный минимализм»

База типографики во всех трёх — **Montserrat**. Философия — `moodboards/Homy-Design-Philosophy.md`. Муд-борды — `moodboards/Homy-Moodboard-A/B/C.png`. Карточка объекта + AI-строка на реальном листинге — `Homy-UI-Mockups.html`.

- **A. Тихий авторитет** (editorial trust). Чернильный синий + бумага + приглушённый «проверочный» зелёный + янтарный «внимание». Montserrat + серифный акцент (IBM Plex Serif) в цифрах. Trust-сигналы — тихие, сбалансированные.
- **B. Тёплый локальный дом** (warm/local). Терракота + крем + олива + абрикос. Montserrat + Lora-акцент. Фото-герой, акцент на районе; человечно, по-армянски, но взросло.
- **C. Интеллект и ясность** (fintech dashboard). Графит + кобальт + сбалансированные зелёный/янтарный сигналы. Montserrat + JetBrains Mono (табличные цифры), командная строка ⌘K. Данные — герой.

Все три сохраняют нейтральное, ненавязчивое доверие покупателя; различается температура: **A — авторитет, B — тепло, C — интеллект.**
